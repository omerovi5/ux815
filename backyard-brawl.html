Backyard Brawl

Party Game Unity Local Multiplayer UI/UX itch.io

Overview

Backyard Brawl is a fast, fun local multiplayer party game built around two completely distinct game modes, each with its own mechanics, visual identity, and strategic depth. The project was developed as part of the Game Design and Development Minor program at Michigan State University.

Game Modes

Candy Collectathon — A Halloween-themed trick-or-treat royale. Players roam a neighbourhood at night, knocking on houses to collect candy. The twist: other players can steal your stash. Race to deposit candy into the witch's cauldron before someone swipes it from you. It's equal parts collection game and heist.

LaserBall — An arcade-style soccer match where goals are scored using laser-powered shots. Power-ups, bouncing shots, and tight arenas make every round unpredictable. Designed for quick rounds that leave players wanting one more game.

My Role

I led UI/UX design and implementation across both game modes. The core challenge was creating two visually distinct experiences — one spooky and atmospheric, one high-energy and sporty — that still felt like they belonged to the same product.

  • Designed mode-specific HUDs: a candy counter and map overlay for Collectathon, a scoreboard and timer for LaserBall
  • Created the main menu, mode select screen, and all settings panels in Figma
  • Implemented all UI elements in Unity using C# and the UI Toolkit
  • Ran internal playtests and iterated on layouts based on player confusion points

Design Highlights

The mode select screen went through several iterations. Early versions buried the mode descriptions, leading to confusion about rules before the game even started. The final design uses a split-screen preview layout, reducing the time to first meaningful game action significantly.

Tools & Tech

  • Unity / C#
  • Figma
  • itch.io deployment
  • Local multiplayer (up to 4 players)
  • Cross-discipline team collaboration
